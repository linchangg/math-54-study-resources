Method of Undetermined Coefficients

Finding a particular solution yp to ay″ + by′ + cy = f(t) by guessing a form and solving for the constants.

Three standard guesses for yp Core rules
Case 1 — Polynomial
f(t) = polynomial
→ try y_p = same degree poly
e.g. f(t)=3t²+1 → try At²+Bt+C
Case 2 — Exponential
f(t) = e^(kt)
→ try y_p = Ae^(kt)
k must be constant. A absorbs the coefficient.
Case 3 — Trig
f(t) = cos(kt) or sin(kt)
→ try Acos(kt)+Bsin(kt)
Always guess both sin and cos even if only one appears in f(t).
Products of these? Use the product of the individual guesses. e.g. f(t) = e−4xsin x → try e−4x(A cos x + B sin x). No extra constant C needed out front — it's redundant.
The three-question checklist Process
1
Is MUC even viable for this f(t)? Only works for polynomials, exponentials, sin/cos, and products/sums of these. tan t, ln t, 1/t → MUC fails.
2
What is the correct form to try for yp? Use the table above, check for overlap (see below).
3
Differentiate yp, plug into the ODE, equate coefficients → solve the linear system for A, B, etc.
The Overlap Problem

The most important nuance — when your guess yp coincidentally satisfies the homogeneous equation.

When overlap occurs
Your guess lives in yc
If your proposed yp is already a solution of the homogeneous equation, plugging it in gives 0 on the left — never equal to f(t). The guess is useless.
Example: y″ + y = cos t
y_c has cos t → overlap!
The fix
Multiply by t (or t²)
Multiply the usual guess by t. If the overlap root is a double root, multiply by t². Same idea as repeated roots in the homogeneous case.
→ try y_p = t(A cos t + B sin t)
⚠
Overlap test: two functions overlap if their ratio is a polynomial. cos t / cos t = 1 ✓ overlap. etsin(3t) / sin(3t) = et ✗ not a polynomial → no overlap. Don't be fooled by shared trig terms when exponentials differ!
Efficiency tips
No redundant constants
When guessing yp = Ce−4x(A cos x + B sin x), the C is redundant — CA is just one constant. Keep it lean; extra unknowns make the algebra longer for no reason.
Don't include homogeneous terms
In y″+y = t cos t, after multiplying by t you get (At²+Bt)cos t + (Ct²+Dt)sin t. Do NOT add extra constant cos t or sin t — those contribute nothing (homogeneous solutions).
Superposition Principle & General Solution

How particular solutions combine, and how to assemble the full general solution.

y(t) = yc + yp
General solution = complementary + particular
Why this works — the superposition principle Theory
If y₁ solves ay″+by′+cy = f₁(t) and y₂ solves ay″+by′+cy = f₂(t), then k₁y₁ + k₂y₂ solves ay″+by′+cy = k₁f₁(t) + k₂f₂(t). This follows directly from linearity of the derivative operator — it behaves well under addition and scalar multiplication, exactly like linear algebra over matrices.
ay″+by′+cy = k₁f₁+k₂f₂
↕
solution: k₁y₁ + k₂y₂
Special case: k₁=1, f₂=0
→ y₂ solves homogeneous
→ this is y_c!
Why the difference of two solutions is homogeneous Key insight
Any two solutions to ay″+by′+cy = f(t) differ by a solution of ay″+by′+cy = 0. This is identical to the matrix equation Ax = b logic: any solution = particular solution + null space. The yc term (with its free constants c₁, c₂) plays the role of the null space — it generalizes the solution.
Step-by-step: full general solution
1
Solve the complementary equation ay″+by′+cy = 0 → get yc = c₁y₁ + c₂y₂
2
Check if there is overlap between yc and f(t) before guessing yp
3
Find any one particular solution yp using MUC or variation of parameters
4
Write y(t) = yc + yp. If initial conditions given, plug in to find c₁, c₂.
Variation of Parameters (B4.6)

A more powerful method that handles f(t) = tan t, ln t, and other functions MUC cannot touch — but requires integrals.

Core idea Intuition
Instead of constants c₁, c₂ in yc = c₁y₁ + c₂y₂, let them become unknown functions u₁(t), u₂(t). Try yp = u₁(t)y₁(t) + u₂(t)y₂(t). When you plug in and simplify, the y₁, y₂ terms cancel (since they satisfy the homogeneous equation), leaving a system of two first-order equations — much easier than the original second-order problem.
yp = u₁(t)·y₁(t) + u₂(t)·y₂(t)
The simplifying assumption & system to solve Setup
We impose the constraint u₁′y₁ + u₂′y₂ = 0 to eliminate u₁″, u₂″ from the equation. This turns the problem into two first-order equations:
u₁′y₁ + u₂′y₂ = 0
u₁′y₁′ + u₂′y₂′ = f(t)/a
Solve via Cramer's Rule. The denominator is the Wronskian W = y₁y₂′ − y₁′y₂.
Explicit formulas via Cramer's Rule Formula
u₁′
u₁′ = −f(t)·y₂ / (a·W)
Then integrate to get u₁(t). No +C needed (we only want one particular solution).
u₂′
u₂′ = f(t)·y₁ / (a·W)
Then integrate to get u₂(t). Same logic — skip the +C.
The Wronskian W = |y₁ y₂; y₁′ y₂′| appears in the denominator. Since y₁, y₂ are linearly independent solutions, W ≠ 0. The denominator is always clean.
⚠
The delicacy warning: VOP works in principle for any f(t), but the integrals ∫u₁′dt and ∫u₂′dt are often impossible or extremely hard. These problems must be carefully designed to make the integrals work out. A slight change (e.g. tan(2t) vs tan(3t)) can make it unsolvable. Integration by parts and trig integrals come up frequently.
Why not always use VOP?
When MUC is better
f(t) is a polynomial, exponential, or trig function → MUC gives a quick algebraic system. No integration needed. Much faster.
When VOP is necessary
f(t) = tan t, sec t, ln t, t⁻¹eᵗ, etc. — functions MUC can't handle. VOP is the only option, but expect messy integrals.
Method Comparison

Quick reference for choosing between MUC and Variation of Parameters.

MUC Method of Undetermined Coefficients
✓ Works for polynomials, ekt, sin/cos, products of these
✓ Pure algebra — no integration
✓ Faster when applicable
✗ Fails for tan, sec, ln, 1/t, etc.
✗ Overlap requires careful t-multiplication
✗ Only constant-coefficient equations
VOP Variation of Parameters
✓ Works for any f(t) in principle
✓ Can handle non-constant coefficients too
✓ No overlap issue to worry about
✗ Requires computing integrals
✗ Integrals often very hard or impossible
✗ Problems must be designed carefully
